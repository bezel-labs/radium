# radium-fonts

Dynamically load the web fonts referenced by your design tokens. Pairs with [`radium`](https://www.npmjs.com/package/radium), which extracts a `FONTS` constant from your tokens; this library injects the matching webfont CDN stylesheet at runtime. Default provider is **Google Fonts**, with a GDPR-friendly **Bunny Fonts** option.

## Install

```sh
npm install radium-fonts
```

## Usage

```ts
import { loadFonts } from "radium-fonts"
import { FONTS } from "@radium/ui/styles/fonts" // token-derived, generated by `radium`

loadFonts([...FONTS])                        // inject Google Fonts <link> + preconnect
loadFonts([...FONTS], { provider: "bunny" }) // use Bunny Fonts instead
```

Call it once at app startup (e.g. in `main.tsx`). It's idempotent — repeat calls with the same fonts do nothing; if the font set changes, the stylesheet is replaced.

Just need the URL?

```ts
import { fontsHref } from "radium-fonts"
fontsHref([{ family: "Montserrat", weights: [400, 600] }])
// "https://fonts.googleapis.com/css2?family=Montserrat:wght@400;600&display=swap"
```

## API

- `loadFonts(fonts, { provider?, display?, preconnect?, document? })` — inject the stylesheet (+ preconnect) into `<head>`. SSR-safe (no-op without a `document`). Returns the `<link>` or `null`.
- `fontsHref(fonts, { provider?, display? })` — build the stylesheet URL.
- `buildHref(provider, fonts, display)` / `preconnectHosts(provider)` — lower-level building blocks.

`FontDef` is `{ family: string; weights?: number[] }`. No framework dependency — pure DOM.
